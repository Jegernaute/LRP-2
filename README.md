# Лабораторна робота: Додаток "Список покупок" (Shopping List)

##  Опис проекту
Цей проект являє собою Android-додаток для ведення списку покупок. Він створений з використанням найсучаснішого стеку технологій розробки під Android: інтерфейс побудовано на **Jetpack Compose**, а для локального збереження даних використовується база даних **Room** у зв'язці з **ViewModel** та **Coroutines**.

## Який функціонал реалізовано?
Базовий функціонал додатку включає:
* **Додавання товарів**: можливість ввести назву нового товару та додати його до списку.
* **Зміна статусу**: можливість відмітити товар як "куплений" за допомогою чекбоксу (Checkbox).
* **Локальне збереження**: всі товари надійно зберігаються в базі даних Room на пристрої, тому список не зникає після закриття додатку або перезавантаження телефону.
* **Стійкість до зміни конфігурації**: завдяки використанню `ViewModel`, екран не втрачає дані під час перевертання пристрою.

##  Додаткові власні доповнення 
Для покращення користувацького досвіду та розширення можливостей додатку було додано наступний функціонал:
1. **Видалення товару**: додано кнопку (іконка кошика) на кожну картку товару для його назавжди видалення з бази даних.
2. **Редагування товару**: реалізовано можливість змінити назву вже існуючого товару. При натисканні на іконку олівця викликається спливаюче вікно (`AlertDialog`), де можна відредагувати текст і зберегти зміни.
3. **Статистика покупок (Лічильник)**: у верхній частині екрану додано динамічний лічильник, який у реальному часі показує кількість вже куплених товарів відносно загальної кількості товарів у списку
    (наприклад: "Куплено: 2 з 5").

##  З якими труднощами зіткнувся та як їх вирішив?
У процесі розробки виникло кілька технічних викликів, переважно пов'язаних з оновленням інструментарію Android Studio:
* **Конфлікти Gradle-плагінів**: Виникала помилка дублювання плагіна Kotlin у нових версіях Android Gradle Plugin (AGP 9.0), оскільки підтримка тепер є вбудованою. Вирішено шляхом очищення блоку `plugins` від
* застарілих викликів.
* **Міграція з KAPT на KSP**: Старий інструмент генерації коду `kapt` (який часто згадується в старих туторіалах для Room) виявився несумісним із новим вбудованим компілятором Kotlin. Проблему вирішено шляхом
* міграції проекту на сучасний та швидший інструмент `KSP` через налаштування файлу `libs.versions.toml`.
* **Робота зі станами в Jetpack Compose**: Виникали труднощі з імпортами делегатів (get/set) для `mutableStateOf` та правильним викликом функцій із `ViewModel` всередині списку `LazyColumn`.
* Було опрацьовано правильну передачу подій (`onClick`) через лямбда-вирази, щоб уникнути нескінченного перемальовування екрану.
* **Підключення іконок Material Design**: Для реалізації додаткового завдання (кнопки видалення та редагування) знадобилося самостійно розібратися,
*  як підключити розширену бібліотеку іконок `material-icons-extended` через систему керування залежностями (Version Catalogs).
